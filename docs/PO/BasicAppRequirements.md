## Historical content

### AAAAT — AgentAgnostic Auto Application Tracker

A handy local tool to track and access candidatures and applications managed by LLM agents or humans.

## Data structure

`*` Might be generated by an LLM from raw content.

`**` Optional, multi-object field could work by pointers to actual artifacts or text blobs.

`***` Pointer to a generated artifact through scripts and LLM processes.

### Candidature

The main logic class. It stores candidature-related information, pending tasks queued for agents and user-assigned todos.

- Candidature ID
- Raw pasted job offer text
- Description*
- Company*
- Company research**
- Role*
- Salary expectation*
- Publication date*
- Application date
- Raw pasted application form
- Application form responses**
- CV sent***
- Cover letter***
- Strengths*
- Things to avoid*
- Questions to ask*
- Tech stack*
- State: draft, sent, precandidate, meeting, closed
- Short pitch*
- Keywords*
- Valuation /100
- Related todos
- Notes

### Tasks

Tasks are queued for agent attention and point to instructions or whatever integration standard is used.

- Task ID
- Task type
- Task instructions
- Task state

### Keywords

Global terms with definitions and user notes. Known terms rendered in candidature content should be selectable.

- Keyword literal
- Admitted forms or aliases
- Definition
- Notes

## Historical UI intent

The UI was originally described as a dynamic single-window dashboard.

### Dashboard states

- `welcomeView`: date, todos, tasks, recent candidatures, summaries, meetings and notes.
- `smartView`: fast recruiter/interview call support with offer context, keywords, suggestions, questions and risks.
- `detailedView`: complete readable and editable candidature fields.
- `userView`: user/profile-oriented editing.

### New candidature input

The early proposal used an expandable panel or modal accepting raw offer content and optional typed fields. It proposed tasks for:

- field inference;
- company research;
- keyword definition;
- CV generation;
- cover-letter generation;
- form-response generation.

### Document generators

The early proposal used LaTeX templates filled programmatically, with sensitive profile values resolved as late as possible.

## Historical user-data intent

AAAAT needs professional profile information to prepare CVs, evaluate opportunities and support career research. The original document proposed transparent privacy classification and a user profile/CV editing section.

All implementation details and release acceptance now follow the authoritative v1 requirements rather than this seed.
