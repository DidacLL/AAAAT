# Sprint 3 Frontend IA Decision Record

## Multi-agent Review Summary

- Information Architect: split the dashboard into fast recognition, a persistent selected-application canvas, and a right inspector instead of a wall of equal panels.
- Interaction Designer: keep server-rendered query navigation with `application_id`, `keyword`, and `tab`; all navigation must work without JavaScript.
- Accessibility Reviewer: use semantic sections, real links for app selection/tabs/keywords, `aria-current` on active rows/tabs, a skip link, and no hover-only critical data.
- Local-first Frontend Engineer: keep standard-library server-rendered HTML, preserve full/read-only/static modes, and add stable `data-*` hooks for future progressive enhancement.
- Agent-workflow Designer: keep raw-offer intake, deterministic review queue, CLI/API/MCP discovery, and provider-free queue rules.
- Contrarian Reviewer: reject the static form/panel direction, remove generated bytecode artifacts, and avoid new dependencies or provider coupling.

## Final Decisions

- Full mode is the normal local working dashboard, not an edit-mode variant.
- The dashboard shell is command/search bar, dense application list, selected application canvas, and right inspector.
- Application rows prioritize company, role, status, next action, last activity, call-probability placeholder, and keyword chips.
- The selected canvas keeps company, role, status, priority, next action, keywords, and short pitch visible while showing contextual inline edit affordances.
- Empty fields use subtle add affordances instead of repeated `Not set` text.
- Inspector tabs are Keyword, Company, Notes, Queue, Artifacts, and Raw intake only in full mode.
- Keyword selection updates inspector context without losing the selected application.
- New opportunities enter through the dedicated raw-offer intake page; the main dashboard does not show a large create form.
- Notes/source links are rendered safely with `target="_blank"` and `rel="noopener noreferrer"` when a URL is present.
- The agent review queue remains deterministic and stored nowhere; it reports raw-offer extraction work, missing review fields, and missing keyword definitions.
